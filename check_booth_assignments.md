# 講師のブース移動問題の調査手順

## 確認項目

### 1. ブラウザのコンソールログを確認
scheduler.htmlを実行後、ブラウザの開発者ツール（F12）を開き、コンソールタブで以下のログを確認してください：

- `📌 初回ブース決定:` - 講師が最初にブースに割り当てられた記録
- `✓ 1日内ブース固定:` - 同じブースが継続使用された記録
- `⚠️ ...既に...に割り当て済み → スキップ` - 優先ブース設定と既存割り当てが衝突した記録

### 2. 1日内ブース割り当ての確認
コンソールの最後に表示される以下のログを確認：
```
=== 割り当て完了 ===
1日内ブース割り当て: Map { ... }
```

このMapに、各講師の日付ごとのブース割り当てが記録されています。

### 3. 優先ブース設定の確認
「④ 講師の優先ブース設定」セクションで、移動している講師に優先ブース設定があるか確認してください。

### 4. 具体的な情報の収集
問題が発生している場合、以下の情報を教えてください：
- 講師名
- 日付
- 移動前のブース番号
- 移動後のブース番号
- その講師の優先ブース設定（あれば）
- コンソールログの該当部分

## よくある原因

### A. 優先ブース設定の競合
講師に優先ブース設定があり、既に別のブースに割り当てられている場合、システムは既存の割り当てを優先します（1日内ブース固定ルール）。

### B. 手動でのドラッグ&ドロップ移動
プレビュー画面で講師をドラッグ&ドロップして手動で移動させた場合、その変更が反映されます。

### C. データの再読み込み
途中でExcelファイルを再読み込みした場合、割り当てが初期化されます。
